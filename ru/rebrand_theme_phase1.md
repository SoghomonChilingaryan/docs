# Rebrand Theme (Phase 1)

Цель фазы 1: безопасно начать rebrand темы без риска сломать логику и без необходимости переписывать весь UI сразу.

## Принципы

- Тема меняется централизованно в `lib/src/common/theme/`.
- Фича-логика не зависит от темы: меняем только presentation (screens/widgets) и общие UI-компоненты.
- UI рефакторим экран за экраном: допускается смешение старого/нового стиля, но не допускаются изменения бизнес-логики.

## Базовый план (миграция по экрану)

1. Добавить/обновить `AppTheme` (светлая/темная) и ключевые токены (surface/outline/primary).
2. Подключить выбор темы в bootstrap через `SettingsBloc` (см. `lib/src/common/core/app/app.dart`).
3. Выбрать 1 экран (например, Player) и привести его к rebrand стилю:
   - не менять количество/назначение кнопок и действий
   - менять только размеры/цвета/отступы/формы/типографику
   - избегать зависимостей между фичами (не импортировать внутренности других фич)
4. Вынести повторяющиеся паттерны в `lib/src/common/widgets/` (кнопки, карточки, pills).

## Что считать "готово" в Phase 1

- Есть стабильная темная тема rebrand и она используется хотя бы на одном экране.
- Настройки позволяют переключить тему/акцент (если нужно).
- Изменения не затрагивают domain/data, только UI.
